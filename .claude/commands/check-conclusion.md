Чеклист проверки данных для документа **Заключение** (`conclusion`, шаблон `conclusion-protocol.docx`).

Применяй ВМЕСТЕ с универсальными проверками типографики и полей из `check.md`.
Корень JSON: `customer / measurementPlace / workplaceCodeNote / totalWorkplaces / measurementDate / rows[] / performer / laboratoryHead / representative`.
Внимание: у этого документа **нет** блока `protocol` и нет номера протокола — проверку «номер протокола» из `check.md` не применяй.

## Обязательные поля (пустая строка = ошибка)
- `customer.name`, `customer.address`
- `measurementPlace`, `workplaceCodeNote`, `totalWorkplaces`
- `measurementDate.day / month / year`
- Минимум одна строка `rows[]`; в каждой строке обязателен `labelRu` и `classValue`
- `labelKk` — **необязательное** (под-строки «мужчины»/«женщины» наследуют казахское название из строки-заголовка выше)
- `performer.fullName/position`, `laboratoryHead.fullName/position`, `representative.fullName/position`

## Формат числовых полей
- `classValue` — из набора **`""`, `"2"`, `"3.1"`, `"3.2"`, `"3.3"`, `"3.4"`, `"4"`** (разделитель — **точка**).
- `count` — целое ≥ 0 **либо** пустая строка `""` (для строк-заголовков без распределения). Не строка с числом.
- `totalWorkplaces` — это **свободная строка с единицей**: `"55 мест"`, а не голое число. Проверь согласование числа и слова.

## Что должно быть подчёркнуто
Вставляемые значения в готовом .docx:
- наименование и адрес заказчика;
- `totalWorkplaces`;
- даты;
- ФИО и должности исполнителя, руководителя лаборатории и представителя.

## Типографика этого документа
- Строки-заголовки факторов заканчиваются двоеточием (`"Тяжесть труда:"`, `"Общая оценка условий труда:"`), а под-строки — `"мужчины"` / `"женщины"` строчными.
- `labelKk` — корректное казахское написание (`"Жарықтандыру"`, `"Микроклиматы"` и т.п.); у под-строк пусто.
- `measurementPlace` — перечисление, в конце может стоять точка; следи за двойными пробелами.
- Месяц — родительный падеж, строчными. `customer.name` — в «ёлочках».
- Сумма `count` мужчин и женщин под заголовком должна совпадать с `count` строки-заголовка.
